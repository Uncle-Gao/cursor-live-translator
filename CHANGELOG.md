# 更新日志 (Changelog)

## [2.5.1] - 2026-04-04

### 🚀 重大突破：Shadow DOM 穿透与全链路鲁棒化升级
- **Shadow DOM 递归探测 (关键)**：重构了 `walkAndTranslate` 核心算法。现已支持递归穿透 Shadow Root，彻底解决了 Agent 侧边栏及深层次隔离组件（如 `jfrog-platform`）无法被汉化探测的问题。
- **API 兼容性加固**：将默认 API 消息角色统一为 `user`。经实测，这解决了官方 API 及部分中转站在处理 Batch 批处理请求时因缺失 `user` 角色导致的连接挂起。
- **稳定性修复**：解决网络抖动或配置错误导致翻译任务卡在 Loading 动画无法消失的问题。
    - **超时机制**：为 OpenAI/DeepL 网络请求增加 12 秒硬性超时控制 (AbortController)。
    - **配置预判**：在发起翻译前校验 API Key 完整性，避免因配置空扫导致的异步挂起。
    - **Webview 桥接优化**：为跨窗口翻译桥接增加 15 秒自清理逻辑。
- **状态机闭环**：修复了“恒等翻译”导致的 Bug（即翻译结果与原文一致时，Loading 动画永不熄灭的问题），实现了“有结果即灭灯”。
- **JSON 解析鲁棒化**：增加正则提取逻辑，支持从带有 Markdown 代码块包裹的 AI 响应中精准抠取翻译字典。
- **内置诊断套件**：暴露 `__I18N_DIAGNOSE__()` 和 `__I18N_TRANSLATE__()` 全局函数。
- **新增屏蔽规则**：默认屏蔽 `[data-resource-name*="."]`，自动识别并保护带有扩展名的文件选项卡。

## [2.5.0] - 2026-04-02

### ✨ 核心突破：Webview 插件深度汉化与全局同步架构
- **插件汉化支持**：正式支持 Cursor 内部 Webview 插件（如 Claude Code）的自动化注入与汉化。
- **跨窗口翻译代理桥 (Translation Bridge)**：通过 `postMessage` 解决了插件 Webview 的 CSP 联网限制问题。
- **通讯穿透增强**：采用 `window.top` 直连与递归广播机制，确保在多层嵌套 iframe 下依然能 100% 触达插件内部。
- **交互升级：全局调试状态同步**：主窗口与所有插件 Webview 的调试高亮（蓝色虚线框）状态现在实时同步，一键联动。
- **分域屏蔽体系**：主窗口与各插件支持独立的屏蔽规则（CSS 选择器、标题、URL）。
- **交互流优化**：
    - 一键汉化后自动询问是否继续汉化插件。
    - 恢复官方流程加入插件联动恢复询问。
- **视觉增强**：翻译中的字段自动追加 Loading 动画，大小与文字高度一致。
- **日志精细化**：在控制台中清晰标注 `[Workbench]` 或 `[Plugin Name]` 的运行状态。
- **多平台构建**：发布 macOS、Windows、Linux 的原生二进制执行文件。

## [2.4.3] - 2026-03-29

### ✨ 交互演进：二级触发与全平台适配
- **二级触发交互**：引入冷门快捷键（Mac: `Cmd+Opt+Shift+B` | Win: `Ctrl+Alt+Shift+B`）动态切换高亮。
- **悬停显示原文 (Alt+Hover)**：按住 `Alt/Option` 悬停于中文上可弹出带“原文：”前缀的浮窗。
- **常态化溯源**：`data-i18n-original` 标签始终注入。
- **一键缓存清理**：安装器新增选项 `5` 支持物理级重置 AI 翻译缓存。
- **彻底卸载引导**：恢复官方流程现允许同时删除本地配置。
- **智能规则合并**：自动将最新屏蔽规则合并至旧版用户配置中。
