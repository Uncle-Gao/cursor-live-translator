# Cursor 汉化 V2 开发计划 (HTML 注入 + DOM 懒翻译)

> 状态：规划中
> 目标：实现基于 MutationObserver 和 HTML 注入的纯运行时免维护汉化方案。

## 概要路径

基于 `v2-dom-injection.md` 的架构设计，V2 版本的开发路径划分为以下五个核心里程碑（Milestones）：

---

### M1阶段：运行时核心模块建立
- **核心目标**：在 electron-sandbox 下编写基础的运行时脚本，完成 DOM 节点的侦听与替换。
- **关键任务**：
  - [ ] 实例化 `MutationObserver`，监听设置面板 DOM 的 `childList`、`subtree` 变化。
  - [ ] 实现针对文本节点（`textContent`）的最简识别与匹配逻辑。
  - [ ] 探查并应对 SolidJS 重渲染带来的性能瓶颈，避免文本闪烁或触发死循环（通过 `requestAnimationFrame` 阻断）。
- **预期交付**：脱离 UI 界面的底层翻译引擎 `cursor-i18n-runtime.js` 初版。

### M2阶段：安装工具的无痛重构
- **核心目标**：彻底丢弃对 `workbench.desktop.main.js` 达几十兆的暴力正则表达式修改，转向通过轻量级 HTML 的注入机制挂载外挂脚本。
- **关键任务**：
  - [ ] 修改 `cursor_zh_pro.js` 工具函数，解析并访问原生的 `workbench.html`。
  - [ ] 在保留原始格式不受损的前提下，向头部动态插入 `<script src="./cursor-i18n.js"></script>` 引用标签。
  - [ ] 重新计算和回填 `product.json` 中的 `checksums` 来规避破损警告机制。
  - [ ] 以 `codesign --force --deep` 对整个 `Cursor.app` 包执行 macOS 重新签名。
- **预期交付**：新版安全且防弹的 `cursor_zh_pro.js` 安装和复原工具。

### M3阶段：动态 API 集成与分级缓存
- **核心目标**：针对未知或新增未记录的英文菜单项，通过外部调用提供托底的“懒在线翻译”能力。
- **关键任务**：
  - [ ] 编写并实现回退和限流（Debounce）处理策略。
  - [ ] 集成 `localStorage` 本地二级持久化缓存结构，避免频发 API 请求，影响前端渲染性能。
  - [ ] 制定缓存的自动化失效协议（在 Cursor 版本升级、指纹发生变更时刷新缓存池）。
  - [ ] 嵌入 MyMemory（免秘钥）或支持自备的 DeepL / Google 开放 API 网关接入。
- **预期交付**：拥有自学习及高命中率功能的综合在线分流模块。

### M4阶段：正则模式映射与动态变量解析
- **核心目标**：满足诸如索引状态、计费次数等携带可变数字和占位符的复合长字符串翻译需求。
- **关键任务**：
  - [ ] 设计和撰写 `terms.json` 面向匹配分组参数结构的结构化定义规范。
  - [ ] 将形如 `(\d+) requests used` 提取变量占位 `$1 次请求已使用` 进而解析和处理。
  - [ ] 完成正则分支在总运行流转中的正确分发与挂载点。
- **预期交付**：能处理复杂排版的 `terms.json` 支持逻辑以及运行时兼容更新。

### M5阶段：全面验收、压力测试和开源发布
- **核心目标**：确保最终用户在绝大部分场景（甚至是极端切换交互下）不会察觉明显的翻译滞后。
- **关键任务**：
  - [ ] 构造高频交互、网络断连或缓存损坏场景下的兜底和压力回放测试。
  - [ ] 整理使用文档、问题追踪答疑并在 GitHub 发布 V2.0.0 正式版包体。
  - [ ] 向用户输出迁移指南（引导此前应用了 V1 的用户平滑清理残留环境）。
- **预期交付**：完备的 Release 包体与 README，达成发布。
