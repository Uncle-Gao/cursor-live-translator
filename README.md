# Cursor-Live-Translator：AI 驱动的实时本地化引擎

> [!NOTE]
> **原项目 `cursor-settings-zh` 已正式更名为 `Cursor-Live-Translator`。**
> 此次更名标志着项目从单纯的设置汉化脚本，进化为支持 AI 实时刷新的全场景本地化引擎。

本项目是专门为 Cursor 定制的高性能实时翻译引擎。通过 **V2.3 Trusted Bootstrap Injection（受信任引导挂载）** 架构，实现了离线字典匹配、动态变量正则解析与多引擎 AI 实时翻译的深度融合，为您提供全方位的中文开发体验。

> [!IMPORTANT]
> **全系架构跨代升级：运行时 DOM 拦截与 AI 异步实时刷新**
> 这是 V2 PRO 架构的革命性突破。为解决传统汉化插件“逢更新必黑屏”且“无法翻译 AI 动态内容”的缺陷，我们采用了外挂系统模式，直接**内联追加至受信任的 `workbench.js` 引导程序**。随后利用 `MutationObserver` 在内存渲染层实时汉化 UI。在最新的 **V2.4.3 版本** 中，我们引入了**二级触发交互 (Dual-Stage Toggle)** 与 **Alt/Option 悬停溯源** 机制，实现了汉化透明度与生产力工具的完美平衡。

## 🌟 核心亮点 (Pro Features)

### 1. 100% 全页面覆盖 (Full-Page Coverage) 🎯
突破传统插件仅能汉化静态菜单的限制：
- **全场景适配**：深度覆盖 **AI 对话流 (Chat)**、**代码报错 (Error Log)**、**侧边栏资源管理器**及所有二级弹出窗口。
- **动态追踪**：无论界面如何跳转，渲染引擎都会实时跟进，实现 100% 的视觉汉化覆盖。

### 2. 进化引擎：新功能自动汉化 (Zero-Day Localization) 🤖
- **动态适配**：当 Cursor 发布新功能或新菜单时，无需等待插件更新。AI 引擎会自动感知并实施翻译。

### 3. 高兼容性：无惧 Cursor 自动更新 (Version-Proof) 🛡️
- **稳定注入**：基于 **V2.3 Trusted Bootstrap Injection** 技术，直接挂载于系统信任链条。即便 Cursor 频繁热更新，汉化处理依然稳如磐石。

### 4. AI 异步实时刷新 (Direct UI Refresh) 🚀
- **无感更新**：当 AI 翻译结果从后台返回时，引擎会自动微调局部 DOM，让原本显示的英文瞬间“跳变”为优雅的中文。

### 5. 精位屏蔽系统 (Refined Shielding) 🛡️
- **代码区隔离**：确保您的每一行代码都保持原生英文，不受汉化引擎任何干扰。
- **V2.4.3 增强：智能屏蔽与合并 (Smart Shielding)**：自动合并最新的 Composer、Agent 侧边栏等官方推荐屏蔽规则，确保 AI 交互区逻辑纯净。

### 6. 交互式溯源：支持全平台快捷键 (Interactive Traceability) 🔍
- **动态高亮**：使用 `Cmd + Option + Shift + B` (Mac) 或 `Ctrl + Alt + Shift + B` (Win) 瞬间切换蓝色虚线边框，定位汉化区域。
- **悬停原词**：开启高亮后，只需按住 **Option** (Mac) 或 **Alt** (Win) 并悬停在中文上，即可弹出带 **“原文：”** 前缀的浮窗。

## 🚀 快速开始

### 1. 环境准备
确保已安装 Node.js 环境。

### 2. 运行安装程序
在终端执行：
```bash
node cursor_zh_pro.js
```

### 3. 操作指引
1. 选择 `3. 配置 AI 翻译引擎` 并录入您的 API Key。
2. 选择 `1. 一键汉化` 并重启 Cursor 即可体验。
3. **快捷键提示**：
    - **Mac**: `Cmd + Opt + Shift + B` (切换高亮) | `Option + 悬停` (查阅原文)
    - **Win**: `Ctrl + Alt + Shift + B` (切换高亮) | `Alt + 悬停` (查阅原文)

## 🛡️ 安全性与隐私说明

1. **API Key 本地化**：存储在本地家目录 `~/.cursor_live_translator/` 之中，不随代码上传。
2. **直连通信**：不设立中转服务器，请求直达官方接口。
3. **安全隔离**：默认跳过所有代码区 (`<pre>`, `<code>`, `.view-lines`) 及输入框，**绝不触碰您的源码**。

## 🛠️ 常见问题 (Troubleshooting)

### 1. 汉化后 Cursor 提示“损坏”？
这是 macOS 的签名保护。请点击安装器菜单中的相应提示，或手动运行：
```bash
sudo xattr -cr /Applications/Cursor.app
sudo codesign --force --deep --sign - /Applications/Cursor.app
```

### 2. 如何确认引擎已启动？
按下 `F12` 查看控制台。若看到紫色印记 `[Cursor-Live-Translator] 动力系统就绪 | V2.4.3`，则代表引擎已开启。
