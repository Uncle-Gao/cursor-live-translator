# Cursor Settings 汉化项目 (cursor-settings-zh)

本项目通过对 Cursor 核心资源文件进行“补丁式”修改，实现设置界面、聊天 UI 及右键菜单的高深度中文化，并提供自动化管理工具。

## 💡 关于项目 (About)
这是一个 **Vibe Coding** 项目。
发起者是一位**设计师**，建立本项目的初衷是为了方便自己深度使用 Cursor，并在汉化这些自定义设置的过程中，帮助更多开发者（无论技术背景）能够无障碍地了解并掌握 Cursor 的各项强大功能。

## 🌟 项目亮点
- **深度汉化**：全方位覆盖设置项，并攻克了由于变量拆分导致的顽固模版字符串。
- **一键式 Pro 脚本**：集成自动探测路径、备份、汉化及校验修复 (Checksum)，告别繁琐流程。
- **免环境分发**：预编译跨平台可执行文件，非技术用户无需安装 Node.js 即可极速汉化。

## 🚀 使用方法

### 方式一：直接运行成品补丁 (推荐)
1. **彻底关闭 Cursor**。
2. 运行 `dist/` 目录下对应系统的程序（如 Windows 下运行 `cursor-zh-pro-win.exe`）。
3. 输入 `1` 执行汉化（程序会自动寻找路径、备份并处理报错）。
4. 重新打开 Cursor 查看效果。

### 方式二：通过 Node.js 运行 (开发者专用)
1. 在根目录下执行：`node cursor_zh_pro.js`。
2. 选择 `1` 执行汉化，或选择 `2` 一键还原官方英文版。

## 🛠️ 常见问题与排障

### 1. 关于“安装损坏”警告 (Installation Corrupt)
> [!IMPORTANT]
> 汉化后 Cursor 若弹出损坏警告，请放心。这是由于哈希指纹变化触发的良性报警。
> - **解决**：本工具已内置自动修复功能。若手动运行，请执行 `node fix_checksum.js`。

### 2. 关于“安装已损坏”警告后无法运行？
如果您汉化后发现 Cursor 无法正常打开，请尝试使用“恢复官方原版”功能，或者检查当前 Cursor 是否有新版本更新。

### 3. 安全性 FAQ
**Q: 运行本工具会降低稳定性吗？**
A: **不会。** 脚本仅针对指定的翻译文件及其哈希声明进行极简替换。Cursor 的核心二进制数字签名与系统级安全校验依然完好无损。
