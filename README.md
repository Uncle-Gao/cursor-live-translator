# Cursor Pro 汉化工具 (cursor-settings-zh)

本项目通过对 Cursor 核心资源文件进行深度适配，实现设置界面、聊天 UI 及右键菜单的高深度中文化。

> [!NOTE]
> **Pro 版核心逻辑**：不再仅仅修改主 JS 文件，而是通过递归扫描 NLS 资源包与自动化校验修复，实现更稳健、覆盖率更高的汉化效果。

## 🌟 项目亮点 (Pro Features)
- **深度防抖汉化**：目前针对极端稳定性进行了调整，聚焦安全精准替换 `workbench.desktop.main.js`，跳过容易引起功能连环失效的冗余插件扫描。
- **全自动防黑屏架构 (Anti-Black-Screen)**：
  - **V8 引擎缓存逃逸**：每次执行汉化或还原后，将会跨平台自动抹除 `CachedData` 与 `Code Cache`，彻底解决因为新老 JS 字节长度不匹配导致的致命白屏。
  - **macOS Gatekeeper 绕过**：内置运行态重签机制，自动执行原生 `xattr` 剥离与临时 `codesign`，无缝突破苹果强制封杀修改版 App 的 JIT 权限锁。
- **防呆式正则防御**：独家引入 `(?=\\s*(?![:=]))` 负向过滤校验，自动精准避开对象 Key 和变量的赋值区操作，从词法层面杜绝了“改动一个词、搞挂全系统”的惨剧。
- **双轨制持久资产库**：汉化内容隔离在 `i18n/` 下。严格区分具有防呆正则保护的泛用短词库（`dictionary.json`）与全局暴力的附加上下文片断库（`fragments.json`）。
- **极客化交互面板 (Dashboard UX)**：执行进度化繁为简。无论经过多少次系统底层重签或依赖包校验，主菜单始终保持清爽的仪表盘样式，告别杂乱的终端日志流。
- **智能安装路径嗅探 (Smart Path Fallback)**：即便用户将 Cursor 安装在非系统盘（如 D 盘），如果在系统环境变量下未检索到默认路径，应用会自动进入交互式容错提问，用户可随意贴入带引号或带冗余空格的自定义路径，脚本会自动纠错补全 `resources/app` 后缀并继续执行。
- **动态版本锚定备份 (Version-Locked Backups)**：备份文件严格双向绑定系统中提取的真版本号（如 `*.[版本号].bak`），彻底杜绝由于 Cursor 官方后台自动热更新后，旧版残留备份被当成新系统的“原版”、错误覆盖从而产生核心程序断代毁灭的致命危机。
- **跨代幽灵废案绞杀 (Orphan Sweeper)**：每次用户启动运行时，应用会自动释放底层探针，在毫秒级内静默全盘扫描并清理那些不属于当前 Cursor 版本时代的古董幽灵备份垃圾，将硬盘与工作区环境保持在最高规格的洁癖标准下。
- **一键全量溯源恢复**：完善的动态双向备份检索，支持一键将所有受影响的资源以及引擎哈希校验位，无损且精准地回档到官方当前的真实底层版本。

## 🛡️ 汉化范围声明与架构哲学 (Scope & Philosophy)

为了追求系统底层的极致稳定，本项目放弃了“狂暴推平全软件”的做法，**特意保留了原生的英文系统级菜单，并严密封锁了第三方依赖插件的汉化**。这是基于我们多次调试得出的绝对安全边界：

1. **为什么坚决不碰原生系统菜单 (`nls.messages.json`)？**
   这是一个体积庞大、多组嵌套的底层 UI 映射表。如果使用外挂程序强制对其执行全局正则替换，极易使 JSON 的 `Key（变量名）` 也遭到污染破坏。最直接的连锁反应是：软件核心装载框架在启动时发现底层变量名挂失，从而发生大面积崩溃罢工，让你连 File 这样的顶级菜单都点不开。
   
2. **为什么屏蔽了庞大的扩展包阵列 (`extensions/**/nls.json`)？**
   官方捆绑内置的 Git 大脑、TypeScript 语言解析服务等高权限组件非常脆弱。一丁点暴力的多语言篡改，都会导致插件在其后台运行态直接死亡，引发最可怕的幽灵 Bug（例如敲代码毫无阻力但再也没有漂亮高亮）。

**💡 我们的妥协与专注：**
我们将全量汉化的火力 100% 精确聚焦到了 `workbench.desktop.main.js` 身上 —— 因为这里才存在着所有 Cursor 自主研发的独占灵魂：**AI 智能体对话系统面板、Composer 多文档编排台、AI 模型配置中心、独占的高阶设置窗口等**。同时享受无门槛的中文 AI 高频功能操作体验与牢不可破的原生底层基盘，是这个工具目前所能提供的最大价值！

## 🚀 使用方法

### 方式一：直接运行成品 (推荐)
1. **彻底关闭 Cursor**。
2. 运行 `dist/` 下对应系统的程序（如 Windows 下运行 `cursor-settings-zh-win.exe`）。
3. 输入 `1` 执行汉化（或输入 `2` 还原）。
4. 重新打开 Cursor 查看效果。

### 方式二：开发者调试 (Node.js)
1. 确保安装了 Node.js 环境。
2. 在根目录下执行：`node cursor_zh_pro.js`。

## 🛠️ 常见问题 (Troubleshooting)

### 1. 为什么我自己加了翻译之后，打开 Cursor 变黑屏死了？
- **绝大多数原因**：如果你是使用自己扩展的字典，往往是因为你往 `i18n/fragments.json` 文件中，抛入了极其普通的 **单词级碎文本**（例如：`All`, `Default`, `Login`, `User`），并且没有为其增加上下文包裹标签（例如没写成 `>"Default"<`）。
- **后果**：`fragments` 是无差别狂暴替换器。这种泛用词汇会导致底层的 JS 渲染组件甚至核心 API 被篡改剥夺（强行造成 Fatal Syntax Error），导致界面进程连同开发者工具瞬间闪崩报废。
- **解决办法**：立即从此文件中剥离危险的英文泛用词，泛用短词只能放在具备正则防御墙的 `i18n/dictionary.json` 之中！

### 2. macOS 下终端提示强制自动签名失败？
由于某些 Mac 系统的本地文件权限问题（比如你没有用管理员权限执行打包程序），自动签名可能会遭到拒绝，此时请关闭 Cursor 并在命令行依次手动执行以下两行代码修复它：
```bash
sudo xattr -cr /Applications/Cursor.app
sudo codesign --force --deep --sign - /Applications/Cursor.app
```

### 3. 汉化后还是有部分英文？
- 汉化结束后往往会有 `missed_terms.log`。请尽量摘取带有前后的标点符号、HTML 标签一并带入 `fragments.json` 协助补全。欢迎提交 **Issue** 或 **PR**。
