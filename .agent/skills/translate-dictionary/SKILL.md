# Cursor 汉化字典维护技能 (translate-dictionary)

## 目的
本技能旨在规范 `translate_cursor.js` 中 `dictionary` 对象的维护过程，确保汉化条目分类清晰、匹配精准且能自适应软件版本的动态更新。

## 字典分类原则 (动态扩展)
字典结构不应死板地固定现有页面名称，而应遵循以下**层级化分类原则**，以便在软件增加新功能或页面时快速扩展：

### 1. 顶层导航层 (Top-Level Navigation)
- **定义**：设置界面最左侧/最顶部的全局分类入口。
- **示例**：`General`, `Agents`, `New Page Name`。
- **操作**：新页面出现时，在此部分添加其标题。

### 2. 页面内容层 (Page Content Hierarchy)
针对每一个具体的设置页面，应按照 UI 的视觉深度进行逻辑分组：

#### A. 核心标题与子标题 (Headers & Sub-headers)
- 对应 UI 中的块标题或分类标签。
- 备注说明所属页面（例如：`=== “常规” 页面 ===`）。

#### B. 设置项标题 (Setting Names)
- 具体的开关、输入框或按钮旁边的名称。

#### C. 设置项描述 (Setting Descriptions)
- 设置项下方较长的解释性文本。
- **重要**：此类文本通常最长，必须严格遵守“长句优先策略”。

### 3. 页面内分组 (Self-Defined Groups)
- 当页面内出现逻辑聚合时（如“白名单”、“隐私保护”），应使用代码注释自发创建子分类，即便它是新加入的。

## 核心维护逻辑
1. **长句子优先 (Longest Match First)**：
   - 在执行替换前，脚本内部必须执行降序排序。
   - **理由**：防止短词误伤长句的结构。

2. **自适应版本更新 (Version Awareness)**：
   - 脚本必须具备检测“官方纯净版”的能力。
   - 如果当前文件包含 0 个汉化条目，则无视旧备份，强制同步当前文件为最新备份。

3. **正则表达式转义**：
   - 所有 Key 必须自动转义正则特殊字符，确保安全性。

## 维护流程
1. **探测新页面**：软件更新后，扫描设置界面发现未汉化的新标题或新分组。
2. **动态插入**：按照上述“层级原则”，在 `dictionary` 中合适的位置（或新建位置）插入翻译。
3. **保持注释一致性**：使用 `// === 页面名 ===` 和 `// 分组名` 保持代码可读性。
4. **验证**：重启 Cursor 并确认新页面中的文案是否正确对齐渲染。
